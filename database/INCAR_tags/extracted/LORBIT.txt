</p>
<p>When <a class="mw-selflink selflink">LORBIT</a> is set, VASP performs a post-processing step of the Kohn-Sham (KS) orbitals to decompose the KS orbitals into local quantum numbers (<span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\displaystyle lm}" xmlns="http://www.w3.org/1998/Math/MathML">

<mstyle displaystyle="true" scriptlevel="0">
<mi>l</mi>
<mi>m</mi>
</mstyle>
</mrow>
<annotation encoding="application/x-tex">{\displaystyle lm}</annotation>
</semantics>
</math></span><img alt="{\displaystyle lm}" aria-hidden="true" class="mwe-math-fallback-image-inline" src="/wiki/index.php?title=Special:MathShowImage&amp;hash=192292e35fbe73f6d2b8d96bd1b6697d&amp;mode=mathml" style="vertical-align: -0.338ex; height: 2.176ex; width: 2.734ex;"/></span>) and obtain local properties, e.g., the on-site charge density or on-site magnetic moments due to the spin degrees of freedom. The decomposition is achieved by means of one of several projection methods selected by <a class="mw-selflink selflink">LORBIT</a>. All these projections rely on the fact that most of the charge density is close to the ion center, and interstitial regions separate them well. This is merely a qualitative approach in contrast to performing a <a class="mw-redirect" href="/wiki/index.php/Wannier_functions" title="Wannier functions">wannierization</a> in order to obtain a localized basis, but often it serves as a good estimate. 
</p>
<p>For VASP version &lt; 6 with <a class="mw-selflink selflink">LORBIT</a> &gt;= 11 and <a href="/wiki/index.php/ISYM" title="ISYM">ISYM</a> = 2, see <a href="/wiki/index.php/Known_issues" title="Known issues">known issues</a>. 
</p>
<h2><span class="mw-headline" id="Projection_methods">Projection methods</span></h2>
<p>See the table for an overview:
</p>
<h3><span id="For_LORBIT_.3C_10"></span><span class="mw-headline" id="For_LORBIT_&lt;_10">For <a class="mw-selflink selflink">LORBIT</a> &lt; 10</span></h3>
<p>The projection is onto spherical harmonics at each ionic site within a sphere defined by <a href="/wiki/index.php/RWIGS" title="RWIGS">RWIGS</a>. The radius must be specified for each atomic species, and there is some uncertainty introduced depending on the size of the sphere.
</p>
<h3><span id="For_LORBIT_.3E.3D_10"></span><span class="mw-headline" id="For_LORBIT_&gt;=_10">For <a class="mw-selflink selflink">LORBIT</a> &gt;= 10</span></h3>
<p>The projection uses the projector functions that are provided by the <a class="mw-redirect" href="/wiki/index.php/PAW_formalism" title="PAW formalism">PAW method</a>. This is, of course, still a qualitative approach because also, for the PAW projectors, the radius was somehow defined, and it is not guaranteed to be the best choice for that particular system as it depends on the chemical composition and crystal or molecular structure.
</p>
<h3><span class="mw-headline" id="Phase_factors">Phase factors</span></h3>
<p>For <a class="mw-selflink selflink">LORBIT</a>&gt;=12:
The <b>phase factors</b> written by VASP can usually only be used as a qualitative measure of the projection of the orbitals into the atomic sphere. The main issue is that most VASP <a href="/wiki/index.php/POTCAR" title="POTCAR">POTCAR</a> files have two or three projectors per <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\displaystyle l}" xmlns="http://www.w3.org/1998/Math/MathML">

VASP combines these two numbers into a single number. The precise algorithms differ in different versions of VASP, and we recommend that you inspect the source code for more details. From vasp.6 onward, an improved scheme has been implemented and can be selected using <a class="mw-selflink selflink">LORBIT</a>=14. In this case, VASP first selects a single projector for each <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\displaystyle l}" xmlns="http://www.w3.org/1998/Math/MathML">

This is done in such a way that the new projector is optimally chosen to represent the calculated orbitals in the energy interval specified by <a href="/wiki/index.php/EMAX" title="EMAX">EMAX</a> and <a href="/wiki/index.php/EMIN" title="EMIN">EMIN</a>. In the second step, VASP projects onto these optimized projectors, yielding a single complex number for each orbital, site and <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\displaystyle l}" xmlns="http://www.w3.org/1998/Math/MathML">

For details we also refer to <sup class="reference" id="cite_ref-Schuler:JPCM:2018_1-0"><a href="#cite_note-Schuler:JPCM:2018-1">[1]</a></sup>.
<a class="mw-selflink selflink">LORBIT</a>=12 should no longer be used except for qualitative calculations. LORBIT=13 chooses the projectors also automatically, but allows for different optimal linear combinations for each orbital.
Note that this is generally not desirable, since the  resultant projection is not compatible with the required properties of a projection operator (a projection operator needs to use energy and orbital independent projectors).
Hence, do not use <a class="mw-selflink selflink">LORBIT</a>=13 for anything but a qualitative analysis.
</p>
<p><a class="mw-selflink selflink">LORBIT</a>=13 and <a class="mw-selflink selflink">LORBIT</a>=14 are only supported by version &gt;=5.4.4. 
</p>
