<p><a class="mw-selflink selflink">ML_IERR</a> = [integer]<br/> Default: <b>ML_IERR</b> = 0 
</p>
<p>Description: Calculation and output frequency of error estimates for <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a>=<i>RUN</i> computations.
</p>
<p>This tag sets the interval in units of molecular-dynamics steps at which the error estimates are written to the <a href="/wiki/index.php/ML_LOGFILE" title="ML LOGFILE">ML_LOGFILE</a>. The error estimate is computed by utilizing the <a href="/wiki/index.php/Machine_learning_force_field:_Theory#Spilling_factor" title="Machine learning force field: Theory">spilling factor</a> if a refit with <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a>=<i>REFIT</i> was done. The <a href="/wiki/index.php/Machine_learning_force_field:_Theory#Spilling_factor" title="Machine learning force field: Theory">spilling factor</a> is calculated for each atom in the current structure and the maximum among all atoms is written to the <a href="/wiki/index.php/ML_LOGFILE" title="ML LOGFILE">ML_LOGFILE</a> file marked with <code>SFF</code>. 
</p>
<p>The <a href="/wiki/index.php/Machine_learning_force_field:_Theory#Spilling_factor" title="Machine learning force field: Theory">spilling factor</a> measures the similarity of the local environment of each atom in the current structure to that of the local reference configurations of the force field. The values of the spilling factor are in the range <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\displaystyle [0,1]}" xmlns="http://www.w3.org/1998/Math/MathML">

If the atomic environment is "properly" represented by the local reference configurations the spilling factor approaches 0. Vice versa the <a href="/wiki/index.php/Machine_learning_force_field:_Theory#Spilling_factor" title="Machine learning force field: Theory">spilling factor</a> quickly approaches 1, if the force field is extrapolating.
</p>
<p>The calculation of the spilling factor scales quadratically with the number of local reference configurations and linearly with the number of species. For force fields containing many species and/or local reference configurations, the evaluation time of the spilling factor becomes of the order of the evaluation of a single force field step or more. Since it is sufficient to monitor the error every  <a class="mw-selflink selflink">ML_IERR</a> MD steps, the total time consumed by the evaluation of the spilling factor can become insignificantly compared to the total time. 
</p>
<p>In long molecular dynamics calculations, we recommend to use at least <a class="mw-selflink selflink">ML_IERR</a>=20-100.
</p>
<p><a class="mw-selflink selflink">ML_IERR</a> can be freely chosen only if <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a>=<i>RUN</i>. In any other calculation mode, if <a class="mw-selflink selflink">ML_IERR</a> is not equal to 1, the code will exit with an error and provide an error description.
</p>
<p>For calculations using force fields obtained by <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a>=<i>REFITBAYESIAN</i> or without any refitting, the Bayesian error estimates of energy, forces and stress (<code>BEFF</code>) are additionally written out to the <a href="/wiki/index.php/ML_LOGFILE" title="ML LOGFILE">ML_LOGFILE</a> file and their output frequency is also controlled by <a class="mw-selflink selflink">ML_IERR</a>. Albeit having the advantage of an additional error estimate we still do not recommend using these force fields, since they are significantly slower than force fields obtained by <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a>=<i>REFIT</i>.
</p>
