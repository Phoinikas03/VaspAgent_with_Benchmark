<p><a class="mw-selflink selflink">ENCUTGWSOFT</a> = [real] 
</p>
<p>Descprition: The flag <a class="mw-selflink selflink">ENCUTGWSOFT</a> sets the energy cutoff for the response function, such that it allows to truncate the Coulomb kernel slowly between the energy
specified by <a class="mw-selflink selflink">ENCUTGWSOFT</a> and <a href="/wiki/index.php/ENCUTGW" title="ENCUTGW">ENCUTGW</a> using a cosine window function.
</p>

For GW type calculations the squeezed Coulomb kernel was the default (when <a class="mw-selflink selflink"><span style="color:black">ENCUTGWSOFT</span></a> was set in the INCAR file) before version vasp.6.3, but
in newer releases the code always defaults to a smoothed Coulomb kernel (both for GW and RPA type calculations). If one desires to recover the behavior
of vasp.6.2 and older versions, <a href="/wiki/index.php/LSCK" title="LSCK">LSCK</a>=.TRUE. must be set in the INCAR file for GW type calculations if <a class="mw-selflink selflink"><span style="color:black">ENCUTGWSOFT</span></a> is set in the INCAR file.
