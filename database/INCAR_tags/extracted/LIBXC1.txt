<p><a class="mw-selflink selflink">LIBXC1</a> = [string] or [integer] 
</p>
<p>Description: <a class="mw-selflink selflink">LIBXC1</a> specifies the exchange or exchange-correlation functional from the library of exchange-correlation functionals Libxc<sup class="reference" id="cite_ref-marques:cpc:2012_1-0"><a href="#cite_note-marques:cpc:2012-1">[1]</a></sup><sup class="reference" id="cite_ref-lehtola:sx:2018_2-0"><a href="#cite_note-lehtola:sx:2018-2">[2]</a></sup><sup class="reference" id="cite_ref-libxc_3-0"><a href="#cite_note-libxc-3">[3]</a></sup>.
</p>
<p><a class="mw-selflink selflink">LIBXC1</a> and <a href="/wiki/index.php/LIBXC2" title="LIBXC2">LIBXC2</a> can be set to a label (string) or number (integer) associated with a functional listed on the Libxc website<sup class="reference" id="cite_ref-libxc_list_4-0"><a href="#cite_note-libxc_list-4">[4]</a></sup>, e.g., <code>GGA_X_PBE</code> and <code>101</code> for PBE exchange. The label indicates if this is an exchange (X), correlation (C), or exchange-correlation (XC) functional, and which family it belongs to, namely LDA (LDA or HYB_LDA), GGA (GGA or HYB_GGA) or meta-GGA (MGGA or HYB_MGGA). If <a class="mw-selflink selflink">LIBXC1</a> corresponds to an exchange functional, then it can be used in combination with <a href="/wiki/index.php/LIBXC2" title="LIBXC2">LIBXC2</a> for the correlation functional.
</p>
<p>Libxc is a separate library package that has to be downloaded<sup class="reference" id="cite_ref-libxc_3-1"><a href="#cite_note-libxc-3">[3]</a></sup> and compiled before VASP is compiled with the corresponding <a href="/wiki/index.php/Precompiler_options#-DUSELIBXC" title="Precompiler options">precompiler options</a> and <a href="/wiki/index.php/Makefile.include#Libxc_.28optional.29" title="Makefile.include">links to the libraries</a>.
</p>
<p>Calculations with Laplacian-dependent meta-GGA functionals and meta-GGA-based hybrid functionals are possible since VASP.6.4.0.
</p>
<h2><span class="mw-headline" id="How_to">How to</span></h2>
<p>The allowed possibilities for <a class="mw-selflink selflink">LIBXC1</a> and <a href="/wiki/index.php/LIBXC2" title="LIBXC2">LIBXC2</a> are the following:
</p>
<ul><li>Both <a class="mw-selflink selflink">LIBXC1</a>  and <a href="/wiki/index.php/LIBXC2" title="LIBXC2">LIBXC2</a> are specified and correspond to exchange and correlation functionals, respectively.</li>
<li>Only <a class="mw-selflink selflink">LIBXC1</a>  is specified and corresponds to an exchange or exchange-correlation functional.</li></ul>
<ul><li><a class="mw-selflink selflink">LIBXC1</a>  and <a href="/wiki/index.php/LIBXC2" title="LIBXC2">LIBXC2</a> can correspond to functionals of different families, e.g., a meta-GGA and a GGA, respectively.</li></ul>
<p>Regarding other tags in <a href="/wiki/index.php/INCAR" title="INCAR">INCAR</a> related to Libxc:
</p>
<ul><li>One also has to specify <a href="/wiki/index.php/GGA" title="GGA">GGA</a> = LIBXC for LDA, GGA and GGA-based hybrid functionals or <a href="/wiki/index.php/METAGGA" title="METAGGA">METAGGA</a> = LIBXC for meta-GGA functionals and meta-GGA-based hybrid functionals. Note that if one of the tags (<a class="mw-selflink selflink">LIBXC1</a>  or <a href="/wiki/index.php/LIBXC2" title="LIBXC2">LIBXC2</a> ) corresponds to a meta-GGA, while the other corresponds to a GGA or LDA, then <a href="/wiki/index.php/METAGGA" title="METAGGA">METAGGA</a> = LIBXC (and not <a href="/wiki/index.php/GGA" title="GGA">GGA</a> = LIBXC) has to be specified.</li>

However, it is still necessary to set <a href="/wiki/index.php/AEXX" title="AEXX">AEXX</a> at the proper value.</li>
