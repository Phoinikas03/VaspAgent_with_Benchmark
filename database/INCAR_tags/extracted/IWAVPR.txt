<p><a class="mw-selflink selflink">IWAVPR</a> = 0 | 1 | 2 | 3 | 10 | 11 | 12 | 13 
</p>
<p>Description: <a class="mw-selflink selflink">IWAVPR</a> determines how orbitals and/or charge densities
are extrapolated from one ionic configuration to the next configuration.
</p>
<p>For <a class="mw-selflink selflink">IWAVPR</a>&lt;10, the file <a href="/wiki/index.php/TMPCAR" title="TMPCAR">TMPCAR</a> is used to store old orbitals that
are required for the prediction. This setting is depreciated, and not supported by the MPI version. 
The recommended settings are <a class="mw-selflink selflink">IWAVPR</a>&gt;10. In this case, the prediction is
performed without an external file <a href="/wiki/index.php/TMPCAR" title="TMPCAR">TMPCAR</a> (i.e. all required arrays
are stored in the main memory).
</p>
<p>The following options are available for <a class="mw-selflink selflink">IWAVPR</a>:
</p>
<ul><li><a class="mw-selflink selflink">IWAVPR</a>=0 no extrapolation, usually not preferable for first-principles molecular dynamics simulations or relaxations of the ions into the groundstate.</li>
<li><a class="mw-selflink selflink">IWAVPR</a>=1|11 Simple extrapolation of the charge density using atomic charge densities (eq. (9.8) in thesis G. Kresse). This switch is convenient for geometry optimizations (ionic relaxation and volume/cell shape with the conjugate gradient or Quasi-Newton methods, i.e. <a href="/wiki/index.php/IBRION" title="IBRION">IBRION</a>=1,2,3 etc.)</li>
<li><a class="mw-selflink selflink">IWAVPR</a>=2|12 A second-order extrapolation for the orbitals and the charge density (eq. (9.9) in thesis G. Kresse) is performed. This results in superior performance for first-principles molecular-dynamics simulations. It might cause instabilities during on-the-fly learning, so the default is 11 in this case.</li></ul>
<ul><li><a class="mw-selflink selflink">IWAVPR</a>=3|13 In this case a second-order extrapolation for the orbitals, and a simple extrapolation of the charge density using atomic charge densities is done. This is a mixture between <a class="mw-selflink selflink">IWAVPR</a>=1 and 2, however, it is usually worse than <a class="mw-selflink selflink">IWAVPR</a>=2.</li></ul>
