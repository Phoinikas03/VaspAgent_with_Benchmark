<p><a class="mw-selflink selflink">ML_ICRITERIA</a> = [integer] 
</p>
<p><br/>
Description: Decides how the error threshold (<a href="/wiki/index.php/ML_CTIFOR" title="ML CTIFOR">ML_CTIFOR</a>) is updated within the machine learning force field method. <a href="/wiki/index.php/ML_CTIFOR" title="ML CTIFOR">ML_CTIFOR</a> determines whether a first-principles calculation is performed.
</p>
<p>The use of this tag in combination with the learning algorithms is described here: <a href="/wiki/index.php/Machine_learning_force_field_calculations:_Basics#Threshold_for_error_of_forces" title="Machine learning force field calculations: Basics">here</a>.
</p>
<p>The following options are possible for <a class="mw-selflink selflink">ML_ICRITERIA</a>:
</p>
<ul><li><a class="mw-selflink selflink">ML_ICRITERIA</a> = 0: The threshold <a href="/wiki/index.php/ML_CTIFOR" title="ML CTIFOR">ML_CTIFOR</a> is not updated. This method is the only method used for <a href="/wiki/index.php/ML_CALGO" title="ML CALGO">ML_CALGO</a>=1. For <a href="/wiki/index.php/ML_CALGO" title="ML CALGO">ML_CALGO</a>=0, this method is only recommended for refining an existing force field. For example, if you know that <a href="/wiki/index.php/ML_CTIFOR" title="ML CTIFOR">ML_CTIFOR</a> has taken a value of 0.03 in previous runs, you can continue to collect training data by now setting the threshold to <a href="/wiki/index.php/ML_CTIFOR" title="ML CTIFOR">ML_CTIFOR</a>=0.03 to capture all contours and areas of the potential energy surface where first-principles data are still missing. To achieve extremely robust force fields, it is recommended to run <a href="/wiki/index.php/NSW" title="NSW">NSW</a>=100000 steps in this mode to slightly above the highest temperature to be considered.</li>
<li><a class="mw-selflink selflink">ML_ICRITERIA</a> = 1: Set <a href="/wiki/index.php/ML_CTIFOR" title="ML CTIFOR">ML_CTIFOR</a> to a value proportional to the average errors of the <a href="/wiki/index.php/ML_MHIS" title="ML MHIS">ML_MHIS</a> steps. <a class="mw-selflink selflink">ML_ICRITERIA</a> = 1, the average is calculated only for errors after updating the force field. Such updates are quite rare, so updates of <a href="/wiki/index.php/ML_CTIFOR" title="ML CTIFOR">ML_CTIFOR</a> are also quite rare in this mode. Furthermore, since the first principle calculations are only performed for configurations with large errors ("outliers"), the force field is updated only after the outliers are taken into account. Therefore, the  errors included in the averaging are typically larger than the average error in this mode.  It is therefore recommended to set <a href="/wiki/index.php/ML_CX" title="ML CX">ML_CX</a> to 0 (default) in this mode.</li>
<li><a class="mw-selflink selflink">ML_ICRITERIA</a> = 2: Update the criteria using the moving average of all previous errors. This method gives the average of the errors of all previous predictions (i.e. all previously considered MD steps), while <a class="mw-selflink selflink">ML_ICRITERIA</a> = 1 gives only the average of the predictions immediately following the retraining. The length of the history in this mode is currently hard-coded and set to 400 steps (or <a href="/wiki/index.php/ML_MHIS" title="ML MHIS">ML_MHIS</a> x 50 in the newer version). This mode tends to continue sampling, and is therefore somewhat prone to oversampling: as errors decrease, the threshold is steadily lowered and additional first-principles computations are initiated. The recommended values for <a href="/wiki/index.php/ML_CX" title="ML CX">ML_CX</a> in this mode are approximately 0.1 to 0.3. = 0,2, a first-principles calculation is typically performed every 50 steps. This means that if the number of ionic steps is, say, <a href="/wiki/index.php/NSW" title="NSW">NSW</a>=50,000, then about 1,000 first-principles calculations should be performed. For many materials, this results in a reasonably good and robust ML database.</li>
<li><a class="mw-selflink selflink">ML_ICRITERIA</a>=3: This mode is the default for reselecting local reference configurations from an existing <a href="/wiki/index.php/ML_AB" title="ML AB">ML_AB</a> file (<a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a> = <i>SELECT</i>). The <a href="/wiki/index.php/ML_AB" title="ML AB">ML_AB</a> file shall contain a <a href="/wiki/index.php/ML_CTIFOR" title="ML CTIFOR">ML_CTIFOR</a> for each structure stored in the <a href="/wiki/index.php/ML_AB" title="ML AB">ML_AB</a> file. These values are used by VASP as error thresholds for structure selection. This also means that the tags <a href="/wiki/index.php/ML_CTIFOR" title="ML CTIFOR">ML_CTIFOR</a>, <a href="/wiki/index.php/ML_CX" title="ML CX">ML_CX</a>, <a href="/wiki/index.php/ML_CSLOPE" title="ML CSLOPE">ML_CSLOPE</a>, <a href="/wiki/index.php/ML_CSIG" title="ML CSIG">ML_CSIG</a> and <a href="/wiki/index.php/ML_MHIS" title="ML MHIS">ML_MHIS</a> set in <a href="/wiki/index.php/INCAR" title="INCAR">INCAR</a> are ignored. This mode is only available when <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a>=<i>SELECT</i> is activated. It is important that the <a href="/wiki/index.php/ML_AB" title="ML AB">ML_AB</a> file contains a <a href="/wiki/index.php/ML_CTIFOR" title="ML CTIFOR">ML_CTIFOR</a> value for each structure included. Otherwise, VASP will throw an error and will also indicate to the user that some <a href="/wiki/index.php/ML_CTIFOR" title="ML CTIFOR">ML_CTIFOR</a> values are missing from the <a href="/wiki/index.php/ML_AB" title="ML AB">ML_AB</a> file.</li></ul>
<p>As mentioned above, the <a href="/wiki/index.php/ML_CX" title="ML CX">ML_CX</a> tag can be used to fine-tune the update of <a href="/wiki/index.php/ML_CTIFOR" title="ML CTIFOR">ML_CTIFOR</a>. 
The fact that the <a class="mw-selflink selflink">ML_ICRITERIA</a> = 1 or <a class="mw-selflink selflink">ML_ICRITERIA</a> = 2 is a matter of taste. Just remember that <a href="/wiki/index.php/ML_CX" title="ML CX">ML_CX</a> must be set differently in both modes.  While <a class="mw-selflink selflink">ML_ICRITERIA</a> = 1, the <a href="/wiki/index.php/ML_CX" title="ML CX">ML_CX</a> = 0.0, <a class="mw-selflink selflink">ML_ICRITERIA</a> = 2, <a href="/wiki/index.php/ML_CX" title="ML CX">ML_CX</a> = 0.2 is a good default.  
Most of our force fields use <a class="mw-selflink selflink">ML_ICRITERIA</a> = 1, but this mode sometimes stagnates and stops the first principle calculations.
On the other hand, and as already mentioned, using <a class="mw-selflink selflink">ML_ICRITERIA</a> = 2 is prone to oversampling, i.e. it may perform too many first principle calculations. 
</p>
