<p><a class="mw-selflink selflink">NOMEGA</a> = [integer] 
</p>
<p>Description: <a class="mw-selflink selflink">NOMEGA</a> specifies the number of (imaginary) frequency and imaginary time grid points.
</p>
<p>Typically <a class="mw-selflink selflink">NOMEGA</a> should be chosen around 50-100 (for the parallel version, <a class="mw-selflink selflink">NOMEGA</a> should be dividable by the number of compute nodes to obtain maximum efficiency). For quick and memory conserving calculations, it is sufficient to set <a class="mw-selflink selflink">NOMEGA</a> to values around <a class="mw-selflink selflink">NOMEGA</a> = 20-30, but then you must expect errors of the order of 20-50 meV for the gap and 100-200 meV for the bottom of the conduction band. </p>
<p>Note that the spectral method (see <a href="/wiki/index.php/LSPECTRAL" title="LSPECTRAL">LSPECTRAL</a>) scales very favorable with respect to the number of frequency points hence <a class="mw-selflink selflink">NOMEGA</a>=30 is usually only slightly faster than <a class="mw-selflink selflink">NOMEGA</a>=100. 
</p>
<ul><li>N.B: Low-scaling GW and RPA/ACFDT calculations require considerably fewer imaginary frequency (and time) points.</li></ul>
