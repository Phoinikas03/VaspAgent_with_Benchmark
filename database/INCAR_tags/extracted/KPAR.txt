<p><a class="mw-selflink selflink">KPAR</a> = [integer]<br/> Default: <b>KPAR</b> = 1 
</p>
<p>Description: <a class="mw-selflink selflink">KPAR</a> determines the number of <b>k</b>-points that are to be treated in parallel (available as of VASP.5.3.2). Also, <a class="mw-selflink selflink">KPAR</a> is used as parallelization tag for <a href="/wiki/index.php/LTMP2_-_Tutorial" title="LTMP2 - Tutorial">Laplace transformed MP2 calculations</a>. 
</p>
<p>VASP currently offers parallelization and data distribution over bands and/or over plane wave coefficients (see <a href="/wiki/index.php/NCORE" title="NCORE">NCORE</a> and <a href="/wiki/index.php/NPAR" title="NPAR">NPAR</a>), and as of VASP.5.3.2, parallelization over <b>k</b>-points.
To obtain high efficiency on massively parallel systems or modern multi-core machines, it is strongly recommended to use all at the same time. Most algorithms work with any data distribution (except for the single band conjugated gradient, which is considered to be obsolete).
</p>
<p>The set of <b>k</b>-points is distributed over <a class="mw-selflink selflink">KPAR</a> groups of compute cores, in a round-robin fashion.
This means that a group of <i>N</i>=(# of cores/<a class="mw-selflink selflink">KPAR</a>) compute cores together work on an individual <b>k</b>-point (choose <a class="mw-selflink selflink">KPAR</a> such that it is an integer divisor of the total number of cores).
Within this group of <i>N</i> cores that share the work on an individual <b>k</b>-point, the usual parallelism over bands and/or plane wave coefficients applies (as set by means of the <a href="/wiki/index.php/NCORE" title="NCORE">NCORE</a> and <a href="/wiki/index.php/NPAR" title="NPAR">NPAR</a> tags).
</p>
<p><b>Note</b>: the data is not distributed additionally over <b>k</b>-points.
</p>
<p><b>Note</b>: <a class="mw-selflink selflink"><span style="color:black">KPAR</span></a> becomes obsolete if <a href="/wiki/index.php/LMP2LT" title="LMP2LT">LMP2LT</a> or <a href="/wiki/index.php/LSMP2LT" title="LSMP2LT">LSMP2LT</a> are set and specifies the number of plane-waves treated in parallel, see <a href="/wiki/index.php/LTMP2_-_Tutorial#Parallelization" title="LTMP2 - Tutorial">here</a> for more information.
</p>
