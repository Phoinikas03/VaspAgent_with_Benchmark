<p><a class="mw-selflink selflink">ML_DESC_TYPE</a> = [integer]<br/> Default: <b>ML_DESC_TYPE</b> = 0 
</p>
<p>Description: This tag selects the descriptor type of the three-body descriptor used in machine learning force fields.
</p>
<p>This tag selects how the three-body descriptor is calculated and the following options are available:
</p>
<ul><li><code style="padding: 2px"><a class="mw-selflink selflink">ML_DESC_TYPE</a> = 0</code>: Standard three-body descriptor. The number of descriptors in the calculations scales quadratically with the number of elements.</li>
<li><code style="padding: 2px"><a class="mw-selflink selflink">ML_DESC_TYPE</a> = 1</code>: Descriptor for which the number of descriptors is linearly scaling with respect to the number of chemical species.</li></ul>

<h2><span class="mw-headline" id="Additional_sparsification">Additional sparsification</span></h2>
<p>All descriptors can be further combined with <a href="/wiki/index.php/Machine_learning_force_field:_Theory#Sparsification_of_angular_descriptors" title="Machine learning force field: Theory">descriptor sparsification</a>   (<a href="/wiki/index.php/ML_LSPARSDES" title="ML LSPARSDES">ML_LSPARSDES</a>=<b>.TRUE.</b>). Generally, the sparsification of descriptors results in a trade-off between accuracy and efficiency. The fraction of descriptors that is kept is specified by <a href="/wiki/index.php/ML_RDES_SPARSDES" title="ML RDES SPARSDES">ML_RDES_SPARSDES</a>. We advise the user to adjust this parameter carefully and test it individually for each system. However, we have experienced the following trends for our test cases: For <a class="mw-selflink selflink">ML_DESC_TYPE</a>=0 a descriptor sparsification of 50 percent <a href="/wiki/index.php/ML_RDES_SPARSDES" title="ML RDES SPARSDES">ML_RDES_SPARSDES</a>=0.5 leaves the accuracy almost untouched. Since <a class="mw-selflink selflink">ML_DESC_TYPE</a>=1 contains already much fewer descriptors than <a class="mw-selflink selflink">ML_DESC_TYPE</a>=0, a 50 percent sparsification for <a class="mw-selflink selflink">ML_DESC_TYPE</a>=1 results in noticeable accuracy loss (additionally to the 5 to 20 percent by the descriptor itself).
</p>
