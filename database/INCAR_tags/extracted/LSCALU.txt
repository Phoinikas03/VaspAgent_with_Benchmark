<p><a class="mw-selflink selflink">LSCALU</a> = [logical]<br/> Default: <b>LSCALU</b> = .FALSE. 
</p>
<p>Description: <a class="mw-selflink selflink">LSCALU</a> switches on the parallel LU decomposition (using scaLAPACK) in the orthonormalization of the wave functions. 
</p>
<p>For <a class="mw-selflink selflink">LSCALU</a>=.TRUE. the LU decomposition in the orthormalization of the wave functions is done in parallel, using scaLAPACK routines.
Provided, of course, <a href="/wiki/index.php/LSCALAPACK" title="LSCALAPACK">LSCALAPACK</a>=.TRUE. and VASP was compiled with scaLAPACK support (<a class="mw-redirect" href="/wiki/index.php/Precompiler_flags" title="Precompiler flags">precompiler flag</a>: -DscaLAPACK).
</p>
<p>In many cases, the scaLAPACK LU decomposition based is <i>slower</i> than the serial LU decomposition (compare the timing <tt>ORTHCH</tt> in the respective <a href="/wiki/index.php/OUTCAR" title="OUTCAR">OUTCAR</a> files). Hence the default is <a class="mw-selflink selflink">LSCALU</a>=.FALSE.
(subspace rotations, however, are still done using scaLAPACK).
</p>
