Usable with <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a> = <i>TRAIN</i>, <i>SELECT</i>, and <i>REFITBAYESIAN</i>.</li>
<li><a class="mw-selflink selflink">ML_IALGO_LINREG</a>=2: QR factorization. Usable with <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a> = <i>REFIT</i> and <i>REFITBAYESIAN</i>.</li>
<li><a class="mw-selflink selflink">ML_IALGO_LINREG</a>=3: Singular value decomposition. Usable with <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a> = <i>REFIT</i> and <i>REFITBAYESIAN</i>.</li>

The regularization can be controlled via <a href="/wiki/index.php/ML_SIGW0" title="ML SIGW0">ML_SIGW0</a>. Usable with <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a> = <i>REFIT</i> and <i>REFITBAYESIAN</i>.</li></ul>
<p>For on the fly learning <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a> = <i>TRAIN</i> and reselection of local reference configurations <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a> = <i>SELECT</i>, it is strictly necessary to use Bayesian regression (<a class="mw-selflink selflink">ML_IALGO_LINREG</a>=1), since uncertainty estimates are only available for Bayesian regression. 
</p>
<p><b>Refitting</b>:
Although the above modes result in an <a href="/wiki/index.php/ML_FFN" title="ML FFN">ML_FFN</a> file that could be used for production runs, we strongly advise to refit the <a href="/wiki/index.php/ML_ABN" title="ML ABN">ML_ABN</a> files. For that copy the <a href="/wiki/index.php/ML_ABN" title="ML ABN">ML_ABN</a> file to the <a href="/wiki/index.php/ML_AB" title="ML AB">ML_AB</a> file and use <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a>= <i>REFIT</i> (if Bayesian error estimation is required during production runs <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a>= <i>REFITBAYESIAN</i> is an option, but at the cost of significatnly slower calculation time). <a href="/wiki/index.php/ML_MODE" title="ML MODE">ML_MODE</a>= <i>REFIT</i> employs <a class="mw-selflink selflink">ML_IALGO_LINREG</a>=4 by default.
</p>
<p>For <a class="mw-selflink selflink">ML_IALGO_LINREG</a>&gt;1, <a class="mw-selflink selflink">ML_IALGO_LINREG</a>=3 and 4 are the most tested approaches and we use <a class="mw-selflink selflink">ML_IALGO_LINREG</a>=4 routinely before employing a machine learned force field. <a class="mw-selflink selflink">ML_IALGO_LINREG</a>=4 gives more stable force fields and better fitting accuracy than <a class="mw-selflink selflink">ML_IALGO_LINREG</a>=3, due to the regularization term employed (for details please see <a href="/wiki/index.php/Machine_learning_force_field:_Theory#Solution_via_regularized_SVD" title="Machine learning force field: Theory">here</a>). 
</p>
<p><a class="mw-selflink selflink">ML_IALGO_LINREG</a>=4 dramatically improves the condition number of the fitting compared to <a class="mw-selflink selflink">ML_IALGO_LINREG</a>=1 since it directly uses the design matrix. In contrast <a class="mw-selflink selflink">ML_IALGO_LINREG</a>=1 requires to use the covariance matrix (square of the design matrix), which effectively doubles the condition number.
However, <a class="mw-selflink selflink">ML_IALGO_LINREG</a>=4 needs significantly more memory than <a class="mw-selflink selflink">ML_IALGO_LINREG</a>=1 (at least twice that much). Please always monitor the memory estimates in the <a href="/wiki/index.php/ML_LOGFILE" title="ML LOGFILE">ML_LOGFILE</a>! It should be also noted that <a class="mw-selflink selflink">ML_IALGO_LINREG</a>=4 is computationally somewhat more demanding than <a class="mw-selflink selflink">ML_IALGO_LINREG</a>=1, but it typically requires between a few minutes and an hour. So usually the extra cost is negligible compared to the original training.
</p>
