<p>The core-level binding energies can be calculated either in the initial-state approximation or the final-state approximation. In the initial-state approximation, a core electron is removed from the core states and added to the valence/conduction bands but no change of the potential (by e.g. relaxation of the valence electrons) is allowed. The core-level binding energy can then be directly calculated by the Kohn-Sham eigenvalues<sup class="reference" id="cite_ref-koehler:prb:04_1-0"><a href="#cite_note-koehler:prb:04-1">[1]</a></sup><sup class="reference" id="cite_ref-lizzit:prb:2001_2-0"><a href="#cite_note-lizzit:prb:2001-2">[2]</a></sup> of the core level <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math alttext="{\displaystyle \epsilon_{c}}" xmlns="http://www.w3.org/1998/Math/MathML">

<p>In the final-state approximation, the electrons (valence electrons in the frozen-core approximation) are allowed to relax, so that the local hole is screened. In other words, a fully self-consistent electronic calculation is carried out with a core hole and an additional electron in the valence/conduction bands.
</p>
<p>The following options are available in VASP:
</p>
<ul><li><a class="mw-selflink selflink">ICORELEVEL</a>=0: The core energies are not calculated (default).</li>
<li><a class="mw-selflink selflink">ICORELEVEL</a>=1: The initial-state approximation is used. This just involves recalculating the KS eigenvalues of the core states</li></ul>
<p>after a self-consistent calculation of the valence charge density. <a class="mw-selflink selflink">ICORELEVEL</a>=1 is a little bit more involved than the calculations using
<a href="/wiki/index.php/LVTOT" title="LVTOT">LVTOT</a>=<i>.TRUE.</i>, since the Kohn-Sham energy of each core state is recalculated. This adds very little extra cost to the calculations. Usually,
the shifts correspond very closely to the change of the electrostatic potential at the lattice sites (calculated using <a href="/wiki/index.php/LVTOT" title="LVTOT">LVTOT</a>=<i>.TRUE.</i>).
</p>
<ul><li><a class="mw-selflink selflink">ICORELEVEL</a>=2: The final-state approximation is used. Electrons are removed from the core and placed into the valence (effectively increasing <a href="/wiki/index.php/NELECT" title="NELECT">NELECT</a>). The VASP implementation excites all selected core electrons for</li></ul>
<p>all atoms of one species. The species, as well as the selected electrons, are specified using
</p>
<p>The electron count <a href="/wiki/index.php/CLZ" title="CLZ"><span style="color:black">CLZ</span></a> specifies how many electrons are excited
from the core. Usually, 1 or 0.5 (Slater's transition state) are sensible choices.
<a href="/wiki/index.php/CLNT" title="CLNT">CLNT</a> selects for which species in the <a href="/wiki/index.php/POTCAR" title="POTCAR">POTCAR</a> file the electrons
are excited. Usually one would like to excite the electrons for
only one atom, this requires changing the <a href="/wiki/index.php/POSCAR" title="POSCAR">POSCAR</a> and <a href="/wiki/index.php/POTCAR" title="POTCAR">POTCAR</a> file,
such that the selected atom corresponds to one species in the <a href="/wiki/index.php/POTCAR" title="POTCAR">POTCAR</a> file.
i.e. if the calculation invokes a supercell with 64 atoms of one type,
the selected atom needs to be singled out, and the <a href="/wiki/index.php/POSCAR" title="POSCAR">POSCAR</a> file will
then contain 63 "standard"  atoms as well as one special  species,
at which the excited core hole will be placed
(the <a href="/wiki/index.php/POTCAR" title="POTCAR">POTCAR</a> file will hold two identical PAW datasets in this case).
</p>
<p>Several caveats apply to this mode.
First, the excited electron is always spherical and multipole splittings
are not available. Second, the other core electrons are not allowed
to relax, which might cause a slight error in the calculated
energies.
Third, absolute energies are not meaningful, since VASP usually
reports valence energies only. Only relative
shifts of the core electron binding energies are relevant
(in some cases, the VASP total energies might become even positive).
</p>
