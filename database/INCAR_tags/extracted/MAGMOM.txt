<p><a class="mw-selflink selflink">MAGMOM</a> = [real array] 
</p>
<p>Description: Initial magnetic moment for each atom if no magnetization density is present. Considered when symmetry is determined.
</p>
<ul><li>For a <b>magnetic calculation from scratch</b> (<a href="/wiki/index.php/ISTART" title="ISTART">ISTART</a>=0), <a class="mw-selflink selflink">MAGMOM</a> specifies (i) the initial on-site magnetic moment for each atom, and (ii) lowers the symmetry of the system (as of VASP.4.4.4). A magnetic calculation could be either a spin-polarized calculation (<a href="/wiki/index.php/ISPIN" title="ISPIN">ISPIN</a>=2) or noncollinear calculation (<a href="/wiki/index.php/LNONCOLLINEAR" title="LNONCOLLINEAR">LNONCOLLINEAR</a>=T). If the <a class="mw-selflink selflink">MAGMOM</a> line breaks a symmetry of the crystal, the corresponding symmetry operation is removed and not applied during the symmetrization of, e.g., charges and forces.</li></ul>
<ul><li>When <b>restarting a magnetic calculation</b>, <a class="mw-selflink selflink">MAGMOM</a> is only used to determine the symmetry of the system and not to set the on-site magnetic moment. Therefore, if you remove the <a class="mw-selflink selflink">MAGMOM</a> tag before restarting from a converged <a href="/wiki/index.php/WAVECAR" title="WAVECAR">WAVECAR</a> or <a href="/wiki/index.php/CHGCAR" title="CHGCAR">CHGCAR</a>, the magnetization is likely to be symmetrized away.</li></ul>
<ul><li><a class="mw-selflink selflink">MAGMOM</a> also specifies the initial on-site magnetic moments when a <b>magnetic calculation</b> (<a href="/wiki/index.php/ISPIN" title="ISPIN">ISPIN</a>=2 or <a href="/wiki/index.php/LNONCOLLINEAR" title="LNONCOLLINEAR">LNONCOLLINEAR</a>=T) is <b>started from a non-spin-polarized calculation</b> (<a href="/wiki/index.php/ISPIN" title="ISPIN">ISPIN</a>=1 and <a href="/wiki/index.php/LNONCOLLINEAR" title="LNONCOLLINEAR">LNONCOLLINEAR</a>=F). This implies restarting with <a href="/wiki/index.php/ICHARG" title="ICHARG">ICHARG</a>=1 while the <a href="/wiki/index.php/CHGCAR" title="CHGCAR">CHGCAR</a> file contains no magnetization density. Starting magnetic calculations from a non-spin-polarized calculation can improve convergence.</li></ul>
<p>The <a href="/wiki/index.php/I_CONSTRAINED_M" title="I CONSTRAINED M">I_CONSTRAINED_M</a> tag can constrain the on-site magnetic moments.
</p>
<h2><span class="mw-headline" id="Format_and_basis">Format and basis</span></h2>
<ul><li>For a spin-polarized calculation (<a href="/wiki/index.php/ISPIN" title="ISPIN">ISPIN</a>=2), <a class="mw-selflink selflink">MAGMOM</a> is a list of NIONS positive or negative values that specify the magnitude and relative orientation of the magnetization on each ion. The on-site magnetic moments have no direction in real space, i.e., no orientation in the lattice.</li></ul>
<ul><li>For noncollinear calculation (<a href="/wiki/index.php/LNONCOLLINEAR" title="LNONCOLLINEAR">LNONCOLLINEAR</a>=T), the on-site magnetic moment is specified by three components for each ion. Without spin-orbit coupling (<a href="/wiki/index.php/LSORBIT" title="LSORBIT">LSORBIT</a>=False), the total energy depends only on the relative direction of the on-site magnetic moments. Hence, you can give the desired magnetic structure in Cartesian coordinates without considering how the lattice matrix or <a href="/wiki/index.php/SAXIS" title="SAXIS">SAXIS</a> is defined.</li></ul>
