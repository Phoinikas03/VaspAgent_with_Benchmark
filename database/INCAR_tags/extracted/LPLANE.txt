<p><a class="mw-selflink selflink">LPLANE</a> = [logical]<br/> Default: <b>LPLANE</b> = .TRUE. 
</p>
<p>Description: <a class="mw-selflink selflink">LPLANE</a> switches on the plane-wise data distribution in real space. 
</p>
<p>For <a class="mw-selflink selflink">LPLANE</a>=.TRUE., the data distribution in real space is done plane wise.
Any combination of <a href="/wiki/index.php/NPAR" title="NPAR">NPAR</a> and <a class="mw-selflink selflink">LPLANE</a> can be used.
</p>
<p>Generally, <a class="mw-selflink selflink">LPLANE</a>=.TRUE. reduces the communication band width during the FFT's, but at the same time it unfortunately worsens the load balancing on massively parallel machines.
<a class="mw-selflink selflink">LPLANE</a>=.TRUE. should only be used if <a href="/wiki/index.php/NGZ" title="NGZ">NGZ</a> is at least 3×(number of nodes)/<a href="/wiki/index.php/NPAR" title="NPAR">NPAR</a>, and optimal load balancing is achieved if <a href="/wiki/index.php/NGZ" title="NGZ">NGZ</a>=<i>n</i>×<a href="/wiki/index.php/NPAR" title="NPAR">NPAR</a>, where <i>n</i> is an arbitrary integer.
</p>
<p>If <a class="mw-selflink selflink">LPLANE</a>=.TRUE. and if the real space projector functions (<a href="/wiki/index.php/LREAL" title="LREAL">LREAL</a>=.TRUE. or ON or AUTO) are used, it might be necessary to check the lines following
</p>
<p>The <tt>max/ min</tt> values should not differ too much, otherwise the load balancing might worsen as well.
</p>
