<p><a class="mw-selflink selflink">LTEMPER</a> = [logical]<br/> Default: <b>LTEMPER</b> = .FALSE. 
</p>
<p>Description: <a class="mw-selflink selflink">LTEMPER</a> specified whether parallel tempering is used. The flag must be used in combination with  <a href="/wiki/index.php/IMAGES" title="IMAGES">IMAGES</a>.
</p>
<p>VASP supports various modes where simultaneous calculations for different INCAR, KPOINTS, POTCAR, or POSCAR files are performed. 
The parallel tempering mode is explained in this section, however, please read the section <a href="/wiki/index.php/IMAGES" title="IMAGES">IMAGES</a> first. Parallel tempering is also known as replica-exchange molecular dynamics.<sup class="reference" id="cite_ref-sugi99_1-0"><a href="#cite_note-sugi99-1">[1]</a></sup>
</p>
<p>If the tags <a href="/wiki/index.php/IMAGES" title="IMAGES">IMAGES</a>=nn and  <a class="mw-selflink selflink">LTEMPER</a>=.TRUE. are set in the <a href="/wiki/index.php/INCAR" title="INCAR">INCAR</a> file, VASP performs parallel tempering calculations. In this case, it is expedient to supply different <a href="/wiki/index.php/INCAR" title="INCAR">INCAR</a> and <a href="/wiki/index.php/POSCAR" title="POSCAR">POSCAR</a> files in each subdirectory 01, 02, 03, ..., nn. For each subdiretory a different simulation temperature should be supplied using the tags  <a href="/wiki/index.php/TEBEG" title="TEBEG">TEBEG</a> in the INCAR files 01/INCAR, 02/INCAR ... nn/INCAR.
</p>

<p>The average acceptance ratios are also written to the <a href="/wiki/index.php/OUTCAR" title="OUTCAR">OUTCAR</a> file. For efficient parallel tempering the acceptance ratio should not fall much below 0.2-0.3. If the acceptance ratio is too small, one usually needs to increase the number of images. However, too many images can also decrease the probability that all images visit all allowed temperatures. 
</p>
