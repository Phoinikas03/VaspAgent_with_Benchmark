<p>For Libxc functionals that are the semilocal component of a hybrid functional, i.e. those with a tag that starts with HYB (<a href="/wiki/index.php/LHFCALC" title="LHFCALC">LHFCALC</a>=.TRUE. will be set automatically if such a functional is selected), the following explains how it works for the mixing and screening parameters:
</p>
<ul><li>Mixing parameter:
<ul><li>It is usually one of the parameters <a class="mw-selflink selflink">LIBXC1_Pn</a>, and can therefore be modified.</li>
<li>For HYB_GGA_XC_PBEH, HYB_GGA_XC_B1WC, HYB_GGA_XC_HSE03, HYB_GGA_XC_HSE06, HYB_GGA_XC_HSE12, and HYB_GGA_XC_HSE12S, the value of <a href="/wiki/index.php/AEXX" title="AEXX">AEXX</a> (even if not specified explicitly in <a href="/wiki/index.php/INCAR" title="INCAR">INCAR</a>) will be used and automatically passed to the corresponding parameter in Libxc. On the other hand, if this corresponding parameter (<a class="mw-selflink selflink">LIBXC1_Pn</a>) is specified in <a href="/wiki/index.php/INCAR" title="INCAR">INCAR</a>, then it will be used (instead of <a href="/wiki/index.php/AEXX" title="AEXX">AEXX</a>), however note that it will be only for the semilocal component of the hybrid functional and not for the exact exchange that will still use <a href="/wiki/index.php/AEXX" title="AEXX">AEXX</a>.</li>
<li>For all hybrid functionals except those listed just above, <a href="/wiki/index.php/AEXX" title="AEXX">AEXX</a> will not be considered for the semilocal component of the hybrid functional, but only for the exact exchange component. Therefore, a particular choice for the mixing parameter has to be done by specifying both <a href="/wiki/index.php/AEXX" title="AEXX">AEXX</a> (for the exact exchange) and the appropriate <a class="mw-selflink selflink">LIBXC1_Pn</a> (for the semilocal component).</li>
<li>The <a href="/wiki/index.php/ALDAX" title="ALDAX">ALDAX</a>, <a href="/wiki/index.php/ALDAC" title="ALDAC">ALDAC</a>, <a href="/wiki/index.php/AGGAX" title="AGGAX">AGGAX</a>, <a href="/wiki/index.php/AGGAC" title="AGGAC">AGGAC</a>, <a href="/wiki/index.php/AMGGAX" title="AMGGAX">AMGGAX</a>, and <a href="/wiki/index.php/AMGGAC" title="AMGGAC">AMGGAC</a> tags are ignored if the Libxc functional is an hybrid functional (those with a tag that starts with HYB).</li></ul></li>
<li>Screening parameter:
<ul><li>It is usually one of the parameters <a class="mw-selflink selflink">LIBXC1_Pn</a> if it is a screened functional, and can therefore be modified.</li>
<li>For HYB_GGA_XC_HSE03, HYB_GGA_XC_HSE06, HYB_GGA_XC_HSE12, and HYB_GGA_XC_HSE12S, the value of <a href="/wiki/index.php/HFSCREEN" title="HFSCREEN">HFSCREEN</a> (even if not specified explicitly in <a href="/wiki/index.php/INCAR" title="INCAR">INCAR</a>) will be used and automatically passed to the corresponding parameter in Libxc. On the other hand, if this corresponding parameter (<a class="mw-selflink selflink">LIBXC1_Pn</a>) is specified in <a href="/wiki/index.php/INCAR" title="INCAR">INCAR</a>, then it will be used (instead of <a href="/wiki/index.php/HFSCREEN" title="HFSCREEN">HFSCREEN</a>), however note that it will be only for the semilocal component of the hybrid functional and not for the exact exchange that will still use <a href="/wiki/index.php/HFSCREEN" title="HFSCREEN">HFSCREEN</a>.</li>
<li>For all hybrid functionals except those listed just above, <a href="/wiki/index.php/HFSCREEN" title="HFSCREEN">HFSCREEN</a> will not be considered for the semilocal component of the hybrid functional, but only for the exact exchange component. Therefore, a particular choice for the screening parameter has to be done by specifying both <a href="/wiki/index.php/HFSCREEN" title="HFSCREEN">HFSCREEN</a> (for the exact exchange) and the appropriate <a class="mw-selflink selflink">LIBXC1_Pn</a> (for the semilocal component).</li></ul></li></ul>
